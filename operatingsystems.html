OS Fundamentals

Core

What is an Operating System?

An OS is system software that manages computer hardware, software resources, and provides common services for computer programs.

What are the main functions of an Operating System?

Core Functions of an OS:

  • Process Management: Creation, scheduling, termination of processes
  • Memory Management: Allocation and deallocation of memory space
  • File Management: Creation, deletion, manipulation of files
  • Device Management: Control of I/O devices through device drivers
  • Security: Protection and access control
  • User Interface: Command-line or graphical interface
  • Networking: Communication between systems
  • Error Detection: Handling hardware/software errors

Types of Operating Systems

Different OS types designed for specific purposes and computing environments.

  • Batch OS: Jobs processed in batches without user interaction
  • Time-Sharing OS: Multiple users share system resources
  • Distributed OS: Manages group of independent computers
  • Real-Time OS: Time-bound responses (Hard/Soft RTOS)
  • Network OS: Manages network resources and security
  • Mobile OS: Optimized for mobile devices (Android, iOS)

Architecture

OS Architecture & Structure

Different architectural approaches to building operating systems.

  • Monolithic: All components in kernel space
  • Layered: Hierarchical layers of functionality
  • Microkernel: Minimal kernel with services in user space
  • Modular: Loadable kernel modules
  • Hybrid: Combination of approaches
Compare Monolithic and Microkernel architectures
Aspect Monolithic Kernel Microkernel
Size Large (all services in kernel) Small (only essential services)
Performance Fast (no context switching) Slower (more context switches)
Reliability Lower (kernel crash = system crash) Higher (service crash ≠ system crash)
Maintainability Difficult (tightly coupled) Easier (modular)
Examples Linux, UNIX Minix, QNX, Mach
Security Potential vulnerabilities Better isolation

Process Concepts

A process is a program in execution with its own address space, resources, and state.

What is a Process Control Block (PCB)? List its components

Process Control Block (PCB): Data structure containing information about a process.

PCB Components:

  • Process ID: Unique identifier
  • Process State: Current state (new, ready, running, etc.)
  • Program Counter: Address of next instruction
  • CPU Registers: Register contents for context switching
  • CPU Scheduling Info: Priority, scheduling parameters
  • Memory Management Info: Base/limit registers, page tables
  • Accounting Info: CPU used, time limits, process number
  • I/O Status Info: List of I/O devices allocated, open files
  • Process Privileges: Access rights to system resources

Synchronization

Process Synchronization

Coordination of processes/threads to maintain data consistency.

Explain the Producer-Consumer problem with semaphore solution

Producer-Consumer Problem: Also called Bounded-Buffer problem, where producer produces items and consumer consumes them from a shared buffer of fixed size.

Requirements:

  • Producer must not produce when buffer is full
  • Consumer must not consume when buffer is empty
  • Only one process can access buffer at a time

Semaphore Solution:

// Shared variables
semaphore mutex = 1; // mutual exclusion for buffer access
semaphore empty = N; // counts empty buffer slots
semaphore full = 0; // counts filled buffer slots

// Producer Process
producer() {
  while(true) {
    produce_item(item);
    wait(empty); // wait if no empty slots
    wait(mutex); // enter critical section
    add_item(item);
    signal(mutex); // exit critical section
    signal(full); // increment filled slots
  }
}

// Consumer Process
consumer() {
  while(true) {
    wait(full); // wait if no items
    wait(mutex); // enter critical section
    remove_item(item);
    signal(mutex); // exit critical section
    signal(empty); // increment empty slots
    consume_item(item);
  }
}

Key Points:

  • wait() decrements semaphore (P operation)
  • signal() increments semaphore (V operation)
  • Order of wait() calls prevents deadlock
  • Binary semaphores (mutex) ensure mutual exclusion
  • Counting semaphores (empty, full) track resource counts

Deadlock

Core

Deadlock Conditions

A deadlock occurs when processes wait indefinitely for resources held by each other.

Explain the four necessary conditions for deadlock

Coffman Conditions (All four must hold simultaneously):

  • 1. Mutual Exclusion:
    • At least one resource must be held in non-sharable mode
    • Only one process can use the resource at a time
    • Example: Printer, tape drive
  • 2. Hold and Wait:
    • Process holding at least one resource is waiting for additional resources
    • Process doesn't release current resources while waiting
    • Example: Process P1 holds R1, waits for R2
  • 3. No Preemption:
    • Resources cannot be forcibly taken from processes
    • Only process holding resource can release it
    • Resources released only voluntarily
  • 4. Circular Wait:
    • Set of processes {P0, P1, ..., Pn} where P0 waits for resource held by P1, P1 waits for P2, ..., Pn waits for P0
    • Forms a circular chain of waiting processes
    • Example: P1 → R1 → P2 → R2 → P1

Deadlock Example:

Process P1: Holds R1, Needs R2

Process P2: Holds R2, Needs R1

→ Circular Wait → Deadlock!

Algorithm

Banker's Algorithm

Deadlock avoidance algorithm that checks for safe states.

Explain Banker's Algorithm with an example

Banker's Algorithm (Dijkstra, 1965): Deadlock avoidance algorithm that tests for safe states before allocating resources.

Data Structures Needed:

  • Available[m]: Available instances of each resource type
  • Max[n][m]: Maximum demand of each process
  • Allocation[n][m]: Currently allocated resources
  • Need[n][m]: Remaining resource needs (Need = Max - Allocation)
// Example: 3 resource types (A,B,C), 5 processes
Available = [3, 3, 2]

// Allocation Matrix
// A B C
P0: 0 1 0
P1: 2 0 0
P2: 3 0 2
P3: 2 1 1
P4: 0 0 2

// Max Matrix
// A B C
P0: 7 5 3
P1: 3 2 2
P2: 9 0 2
P3: 2 2 2
P4: 4 3 3

// Need Matrix (Max - Allocation)
// A B C
P0: 7 4 3
P1: 1 2 2
P2: 6 0 0
P3: 0 1 1
P4: 4 3 1

Safety Algorithm Steps:

  1. Initialize Work = Available, Finish[i] = false for all i
  2. Find process i where Finish[i] = false and Need[i] ≤ Work
  3. If found: Work = Work + Allocation[i], Finish[i] = true, repeat step 2
  4. If all processes have Finish[i] = true, system is in safe state

Resource Request Algorithm:

  1. If Request[i] ≤ Need[i], go to step 2, else error (exceeded max claim)
  2. If Request[i] ≤ Available, go to step 3, else wait (resources not available)
  3. Pretend to allocate: Available = Available - Request[i], Allocation[i] = Allocation[i] + Request[i], Need[i] = Need[i] - Request[i]
  4. Run safety algorithm; if safe, allocation done; else, rollback

OS Fundamentals & Architecture (12 Questions)

What are system calls? Explain different types with examples

System Calls: Programming interface to services provided by OS kernel.

Types of System Calls:

  • Process Control:
    • create_process(), terminate_process()
    • wait(), signal(), fork(), exec(), exit()
  • File Management:
    • create_file(), delete_file(), open(), close()
    • read(), write(), seek(), get_attributes()
  • Device Management:
    • request_device(), release_device()
    • read_device(), write_device()
  • Information Maintenance:
    • get_time(), set_time()
    • get_system_data(), set_system_data()
  • Communication:
    • create_message_queue(), send_message(), receive_message()
    • shared_memory_create(), shared_memory_attach()
  • Protection:
    • set_permission(), get_permission()
    • allow_user(), deny_user()
// Example: fork() system call in C
#include <stdio.h>
#include <unistd.h>

int main() {
  pid_t pid = fork();
  if (pid == 0) {
    printf("Child Process\n");
  } else if (pid > 0) {
    printf("Parent Process, Child PID: %d\n", pid);
  } else {
    printf("Fork failed\n");
  }
  return 0;
}

Operating Systems Glossary

PCB

Process Control Block - Data structure containing process information.

Context Switch

Saving state of current process and loading state of another.

Thrashing

Excessive paging causing low CPU utilization.

Semaphore

Synchronization variable used for process coordination.

Mutex

Mutual exclusion lock for protecting critical sections.

Page Fault

Interrupt when needed page is not in memory.

TLB

Translation Lookaside Buffer - Cache for page table entries.

RAID

Redundant Array of Independent Disks - Data storage technology.

System Call

Programmatic way for program to request OS services.

Kernel

Core component of OS managing system resources.

Shell

User interface for accessing OS services.

Daemon

Background process providing system services.
